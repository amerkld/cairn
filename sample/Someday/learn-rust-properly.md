# Learn Rust properly

I can read Rust. I cannot *write* Rust from a blank file without stumbling over the borrow checker every 30 seconds. That's the gap.

## What "properly" means

Not passing the Rustlings exercises — I did those. Properly means:

1. Building one non-trivial thing end-to-end (CLI tool, ~2,000 LOC).
2. Understanding lifetimes well enough to design APIs, not just follow the compiler's suggestions.
3. Knowing which async runtime I'd reach for and why.

## Candidate project

A **markdown-to-OPML converter** for outlining tools. Small enough to finish, interesting enough to keep me going, and the domain is comfortable.

## Resources

- *Programming Rust* — the O'Reilly book. Have it already.
- Jon Gjengset's YouTube — deep dives. 30 min chunks.
- The Rust Reference for actually understanding what the book hand-waves over.
