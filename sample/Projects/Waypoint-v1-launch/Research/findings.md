# Research findings

Rolling synthesis across all beta interviews and in-app feedback. Updated when new interviews happen.

## Strong signal (≥10 independent mentions)

1. **Offline reliability is the product.** Not a feature, the product.
2. **Two distinct user segments** exist: thru-hikers (deep, technical, small) and weekenders (broad, simple, large).
3. **Search is quietly broken** — literal prefix match only, no normalization.

## Moderate signal (5–9 mentions)

- Share-by-link for friends who haven't installed the app
- Group saved routes by region / area
- Better elevation profile readability (zoom)

## Weak signal (<5 mentions)

- Apple Watch companion
- Strava import
- Print-friendly route cards
- Dark-mode map tiles

## What changes because of this

| Finding                        | v1 action                          | Post-v1 action               |
| ------------------------------ | ---------------------------------- | ---------------------------- |
| Offline = the product          | Lead the launch copy with it       | —                            |
| Two segments                   | Neutral hero language              | Dedicated onboarding flows   |
| Search is broken               | Normalize queries + accents        | Fuzzy match, rank by recency |
| Share-by-link                  | *Ship it.* Static map page.        | Inline embed for blogs       |
| Region grouping                | —                                  | Saved-route view overhaul    |
